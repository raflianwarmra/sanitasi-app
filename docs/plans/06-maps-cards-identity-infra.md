# Plan 06: Map Correctness, Priority Cards, Bolder Identity, Infra Detail

Per PM brief "Correct Geographic Maps and Improve Dashboard Interactivity".
Priority order is mandated: maps first, validated, before anything else.

## 1. Geographic correctness (highest priority)

**Architecture note:** this dashboard does not ship static SVG files with
path IDs. Geometry is generated by `scripts/build_geo.py` /
`build_national_geo.py` into `public/geo/*.json` and rendered at runtime by
`ProvinceKabkotMap`, joined to data by BPS kode (name fallback). So "audit
every SVG" translates to: audit the **geometry→kode assignment** in the
build pipeline. Labels, tooltips, hover, click routing all derive from the
kode join — fix the assignment and the interaction layer follows.

**Root cause identified:** the builder matches by kode FIRST against two
sources; source 1 (marifauzan/BIG) uses **kemendagri** codes, the sheet
uses **BPS** codes. They collide in the 9xxx range (e.g. kemendagri 92.01 =
Manokwari/Papua Barat vs BPS 9201 = Raja Ampat/Papua Barat Daya), so
kode-first matching can assign the wrong shape — precisely in the Papua
provinces the brief singles out. Source 2 (azunzios/GADM `CC_2`) is
old-BPS-coded and safe for kode matching.

**Fix:**
- Guarded matcher in both build scripts: a kode match is accepted ONLY if
  the source feature's province name or kab name also agrees (normalized);
  otherwise fall through to name-based matching. Applies to both sources
  (cheap insurance on GADM too).
- Regenerate all 38 `prov-XX.json` + `indonesia-provinces.json`.
- New `scripts/validate_geo.py` — systematic, all-regions validation:
  1. coverage: every one of the 514 sheet kode has exactly one feature, in
     the correct province file (kode prefix);
  2. no duplicates / no foreign kode in any file;
  3. **centroid coherence**: each kab centroid vs its province's centroid;
     flag outliers (> 3.5× median distance) — catches any residual
     cross-province shape assignment;
  4. per-province feature-count vs sheet-count table for all 38 incl. the
     six Papua provinces.
  Report written to `docs/geo-validation.md` and required to pass before
  the UI work ships.
- rezkyyayang/maps is used as the visual cross-reference for spot-checking
  flagged regions (its Papua files predate pemekaran, per the brief, so the
  six Papua provinces are validated against kab-level composition from the
  BPS kode groupings — which is exactly how our national dissolve already
  constructs them).

## 2. Isu Prioritas → three interactive horizontal cards

Provinsi page:
- Three cards: Tanpa IPLT (critical), BABS terbuka > 10% (critical), Akses
  aman < 5% (warning). Hierarchy per card: big count → category → threshold
  line → kab/kota list.
- Desktop: 3-across grid. Below ~900px: horizontal scroll-snap strip with
  partial next-card visibility (discoverability) — no compression.
- Every kab/kota entry is a real button → navigates to that profile
  (`/kabkota?provinsi=…&kode=…`); hover/focus states; 44px touch rows.
- "Unit tidak berfungsi" (4th existing alert) is NOT removed — kept as a
  compact strip below the three cards (brief: preserve all information).
